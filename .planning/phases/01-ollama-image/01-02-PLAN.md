Create `.github/workflows/build-ollama.yml` with the following exact structure:

1. Workflow header comment block (follow build-container.yml style with full-width separators):
   - Builds Ollama image with pre-baked gemma4:26b and gemma4:e4b
   - Per-model GHA cache scopes prevent 30+ min rebuilds when only one model changes
   - Validates both models via /api/tags before publish
   - Publishes :latest and :sha-{7char} to GHCR on master push

2. Workflow name: "Build and Publish Ollama Image"

3. Triggers (D-04 — path-scoped, manual trigger included):
   on:
     push:
       branches: [master, 'feature/**']
       paths:
         - Dockerfile.ollama
         - .github/workflows/build-ollama.yml
     pull_request:
       paths:
         - Dockerfile.ollama
         - .github/workflows/build-ollama.yml
     workflow_dispatch:   # manual trigger for model version bumps

4. Job 1 — lint:
   name: "Lint (ShellCheck)"
   runs-on: ubuntu-latest
   steps:
     - uses: actions/checkout@v4
     - name: Install ShellCheck
       run: sudo apt-get install -y -qq shellcheck
     - name: Note (no shell scripts in this phase)
       run: echo "No .sh files in Dockerfile.ollama scope — lint job is a pass-through placeholder for workflow_dispatch and future scripts"

5. Job 2 — build-and-test:
   name: "Build and Test"
   runs-on: ubuntu-latest
   needs: lint
   permissions:
     contents: read
     packages: write

   Steps in this exact order:
   a. Maximize build disk space (MUST BE FIRST — before checkout):
      uses: easimon/maximize-build-space@master
      with:
        remove-dotnet: 'true'
        remove-android: 'true'
        remove-haskell: 'true'
        remove-codeql: 'true'
        remove-docker-images: 'true'
        root-reserve-mb: 2048
      # Comment: Must be first — reformats build volume; cannot run after checkout

   b. uses: actions/checkout@v4

   c. uses: docker/setup-buildx-action@v3

   d. Build image (per D-06 — per-model cache scopes, mode=max):
      name: "Build image"
      uses: docker/build-push-action@v5
      with:
        context: .
        file: Dockerfile.ollama
        push: false
        load: true
        tags: ollama-models:ci
        cache-from: |
          type=gha,scope=ollama-gemma4-26b
          type=gha,scope=ollama-gemma4-e4b
        cache-to: |
          type=gha,mode=max,scope=ollama-gemma4-26b
          type=gha,mode=max,scope=ollama-gemma4-e4b

   e. Validate models present (Pattern 5 — curl from runner, not inside container):
      name: "Validate models present"
      run: |
        docker run --rm -d --name ollama-ci \
          -e OLLAMA_HOST=0.0.0.0:11434 \
          -p 11434:11434 \
          ollama-models:ci
        echo "Waiting for Ollama server to be ready..."
        for i in $(seq 1 30); do
          curl -sf http://localhost:11434/api/tags >/dev/null 2>&1 && break
          echo "  attempt $i/30..."
          sleep 3
        done
        TAGS=$(curl -sf http://localhost:11434/api/tags)
        echo "$TAGS" | grep -q "gemma4:26b" || (echo "FAIL: gemma4:26b not found in /api/tags" && docker stop ollama-ci && exit 1)
        echo "$TAGS" | grep -q "gemma4:e4b"  || (echo "FAIL: gemma4:e4b not found in /api/tags"  && docker stop ollama-ci && exit 1)
        echo "PASS: Both models present in /api/tags"
        docker stop ollama-ci

6. Job 3 — publish:
   name: "Publish to GHCR"
   runs-on: ubuntu-latest
   needs: build-and-test
   if: github.ref == 'refs/heads/master' && github.event_name == 'push'
   permissions:
     contents: read
     packages: write   # T-1-02 mitigation: scoped to packages only

   Steps in this exact order:
   a. Maximize build disk space (same config as build-and-test job — MUST BE FIRST)

   b. uses: actions/checkout@v4

   c. uses: docker/setup-buildx-action@v3

   d. Log in to GHCR:
      uses: docker/login-action@v3
      with:
        registry: ghcr.io
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}

   e. Set lowercase owner and short SHA (D-03):
      name: "Set lowercase owner and short SHA"
      run: |
        echo "OWNER=$(echo '${{ github.repository_owner }}' | tr '[:upper:]' '[:lower:]')" >> $GITHUB_ENV
        echo "SHORT_SHA=${GITHUB_SHA::7}" >> $GITHUB_ENV

   f. Note GHCR layer limit (D-09):
      name: "Note GHCR multi-layer push"
      run: |
        echo "INFO: gemma4:26b (~18GB) and gemma4:e4b (~9.6GB) exceed GHCR's 10GB per-layer limit."
        echo "INFO: Docker push client will split layers automatically — this is expected behavior."

   g. Push to GHCR (D-03 tags, D-06 cache scopes):
      name: "Push to GHCR"
      uses: docker/build-push-action@v5
      with:
        context: .
        file: Dockerfile.ollama
        push: true
        tags: |
          ghcr.io/${{ env.OWNER }}/dotfiles/ollama-models:latest
          ghcr.io/${{ env.OWNER }}/dotfiles/ollama-models:sha-${{ env.SHORT_SHA }}
        cache-from: |
          type=gha,scope=ollama-gemma4-26b
          type=gha,scope=ollama-gemma4-e4b
        cache-to: |
          type=gha,mode=max,scope=ollama-gemma4-26b
          type=gha,mode=max,scope=ollama-gemma4-e4b
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → GHCR | GITHUB_TOKEN used to authenticate push to ghcr.io |
| GHA cache store → build jobs | Cache entries restored from repository-scoped GHA cache during build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-02 | Information Disclosure | GITHUB_TOKEN in workflow permissions | mitigate | Scope `permissions` to `packages: write` + `contents: read` only — no `secrets: read`, no `actions: write`, no `contents: write`. Enforced in acceptance criteria (grep for `contents: write` must exit 1). |
| T-1-03 | Tampering | GHA cache restore (model layer cache) | accept | GHA cache is repository-scoped; only collaborators with repository write access can write cache entries. Risk accepted; no additional mitigation needed for this threat level. |
</threat_model>
